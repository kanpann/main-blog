# 책 외부

일단 내가 책을 고르는 기준으로는

- 책이 흐물거리지 않는 재질인가
- 그립감이 괜찮은가
- 가독성이 좋은가
- 책 표지 재질이 튼튼한가
- 기승전결이 괜찮은가(목차 구성)

기준들을 보면 재질이나 그립감 가독성 등을 중요하게 생각하는데 은근히 이런 것들이 좋은 책들이 좋은 책인 경우가 더 많았고 한 번이라도 더 읽게 되기 때문이다. 그런 점에서 이 책은 내 기준에서는 충분히 합격점이라고 볼 수 있다.

# 책 내부

## 전체적인 느낌

입문서에 걸맞게 js(es6)부터 Node.js 기초부터 설명하는 점은 개인적으로 마음에 들었다. 그리고 예제 중심으로 설명을 곁들여 실무에서 사용하는 기술 위주로 설명하고 있는데, 한번 다른 리뷰들도 찾아보았는데 이 부분이 케바케인 것 같다. 기존에 Node.js를 조금 사용해본 사용자들은 다른 책보다 좋다고 하기도 하고, js 입문자들의 경우 숨이 턱턱 막힌다고 표현하는 것을 보았는데 내 기준에서 보았을 때는 어렵지 않게 필요한 부분들로 구성해놓은 것 같았다. Node.js를 조금 사용해본 내 기준에서는 "Node.js 참고서" 같은 느낌이 더 크게 느껴졌던 것 같다.

## 목차 구성에 대해

목차 구성은 잘해놓은 것 같은데 다른 사람들과 같이 나 역시도 REST API에 대한 목차가 조금 빈약한 것이 아쉬웠다. Node.js(Express)로 템플릿 엔진을 구성해서 풀스택으로 사용하는 경우도 많이 있지만 프론트엔드 프레임워크들과 곁들여서 백엔드로 사용하는 경우가 점차 늘고 있는데, 이런 내용은 조금 다른 내용에 비해 비중이 적은 느낌이 있다.
근데 이에 대해서도 반론하자면 사실 어쩔 수 없는 부분이기도 하다. 이 책은 Node.js(Express)에 대한 책이지 REST API도 비중 있게 다루려고 하면 필연적으로 프론트에 대한 설명도 들어가야 하고, 그러다 보면 책 내용이 길어지기 때문이다. 그렇다고 REST API 구현에 대한 내용만을 다루다 보면 입문자 입장에서는 화면은 없고, 코드만 작성하게 되니 흥미를 잃을 수 있기 때문에 충분히 이해되는 부분이다.

# 총평

이 책은 충분히 입문자가 보기에 충분한 책이고, Node.js(Express)로 구현하는 실무 위주의 코드를 제공하는 좋은 책임에 틀림이 없는 것 같다. 그리고 Node.js 책의 경우 선택지가 많이 없기도 하다🤣

책에 대한 자세한 설명은 다른 분들이 많이 해놓기도 했고, 책 소개로도 충분히 짐작이 가능하기 때문에 전체적인 느낌 위주로 한번 작성해보았다. 당연히 Node.js 책이기 때문에 JavaScript를 잘 모르는 사람의 경우 어려울 수도 있을 것 같지만, 기존에 JavaScript를 조금 다뤄본 사람이라면 충분히 재미있게 읽으며 ⭐️ 5개를 줄 수 있을 것 같다.
